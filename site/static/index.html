tinyhttp is a modern, lightweight and modular Express-like web framework for Node.js. 

pnpm add @tinyhttp/app @tinyhttp/logger
import { App } from '@tinyhttp/app'
import logger from '@tinyhttp/logger'

const app = new App()

app
  .get('/', (_, res) => void res.send('Hello World'))
  .get('/page/:page/', (req, res) => {
    res.status(200).send(`
    Some cool page
    URL
    ${req.url}
    Params
    ${JSON.stringify(req.params, null, 2)}
  `)
  })
  .use(logger())
  .listen(3000, () => console.log('Started on http://localhost:3000'))
http://localhost:3000/

Features

Same API without legacy dependencies

tinyhttp uses a bare minimum of dependencies to avoid shipping useless ES6 polyfills (that Express has). node_modules is smaller in size and code executes faster. Also, it targets ES2019 (Node 12.4+) that allows to use modern syntax to ship as less code as possible.

Compatible with Express

All middleware for Express works the same for tinyhttp. Explore examples with using Express-oriented middleware.

Premade popular middleware

tinyhttp offers a list of popularly used middleware as separate packages. See full middleware list.

Compatible with Deno [not yet]

tinyhttp is planned to be available as a Deno module [soon]

Completely modular

Every piece of tinyhttp is published separately as a package that you can use outside of tinyhttp. For example, you can use these modules with built-in `http` module.

Async support

In comparison to Express, tinyhttp handles async / await without any extra setup with keeping a proper middleware handling loop.
