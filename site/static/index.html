tinyhttp is a modern, lightweight and modular Express-like web framework for Node.js. 

pnpm add @tinyhttp/app @tinyhttp/logger
import { App } from '@tinyhttp/app'
import logger from '@tinyhttp/logger'

const app = new App()

app
  .get('/', (_, res) => void res.send('Hello World'))
  .get('/page/:page/', (req, res) => {
    res.status(200).send(`
    Some cool page
    URL
    ${req.url}
    Params
    ${JSON.stringify(req.params, null, 2)}
  `)
  })
  .use(logger())
  .listen(3000)
http://localhost:3000/

Some cool page

/page/test

Features

Same API without legacy dependencies

tinyhttp uses a bare minimum of dependencies to avoid shipping useless ES6 polyfills (that Express has). node_modules is smaller, code runs faster, everyone wins! Also, it targets ES2019 (Node 12.4+) to ship as less code as possible.

Compatible with Express

tinyhttp offers a pack of popular middleware like logger and cookie parser but all middleware for Express works for tinyhttp as well.

Completely modular

Every piece of tinyhttp, from static server to eTag is published separately as a module that you can use outside of tinyhttp.
